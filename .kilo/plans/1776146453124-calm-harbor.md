# Plan: Fix Email Verification Issue

## Problem
- With `MAIL_MAILER=log`, emails are written to log file, not sent to user
- User can't receive verification link to click
- User can't login because email is not verified

## Root Cause
The register flow requires email verification before login works (line 167 in AuthController.php), but with log driver there's no way to deliver the verification link.

## Solution

**Approach: Auto-verify email on registration**

Modify the register function in `be/app/Http/Controllers/API/AuthController.php` to set `email_verified_at` immediately when creating the user.

### Implementation

1. In `register()` function (around line 41-53), add `email_verified_at => now()` when creating the user:
   ```php
   $user = User::create([
       ...
       'email_verified_at' => now(),  // Auto-verify for now
   ]);
   ```

2. Also update the success message to reflect that email is already verified:
   ```php
   'message' => 'Registrasi berhasil. Email Anda sudah terverifikasi.',
   ```

This is the simplest fix - new users are verified immediately upon registration.

## Alternative (if email verification is required later)

Add an environment variable to toggle email verification:
- Set `REQUIRE_EMAIL_VERIFICATION=false` in Railway variables to skip verification
- Keep it `true` (default) when ready to enable real email verification
